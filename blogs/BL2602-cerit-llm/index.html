AI · Benchmark · CERIT

Claude Code na české akademické infrastruktuře: co ukázal benchmark 600 běhů

Michal Cifra · ÚFE AV ČR / Biophotoniq · testy: duben 2026 · ~2 200 slov

CERIT-SC z e-INFRA CZ provozuje LLM gateway zdarma pro českou akademii — a to s rozhraním kompatibilním s Anthropic API. Přesměrovat na ni Claude Code je otázka jednoho řádku v terminálu. Testoval jsem, co to v praxi znamená: spustil jsem přes 600 agentických běhů na 10 reálných vývojářských úlohách a porovnal výsledky s nativním Claude. Tady je co z toho vzešlo.

Jak to celé funguje

Normálně Claude Code volá Anthropic servery. Jenže CERIT nabízí endpoint (https://llm.ai.e-infra.cz/), který mluví stejným protokolem — stačí nastavit proměnnou prostředí ANTHROPIC_BASE_URL a Claude Code začne posílat dotazy na českou infrastrukturu místo do Ameriky. Model na druhé straně může být úplně jiný.

Aby to bylo smysluplné porovnání, navrhl jsem čtyři různé způsoby volání, které jsem v benchmarku nazval tracky:

TrackCo se dějePočet běhů
A Nativní Claude Code s Anthropic API — zlatý standard 100
B CERIT modely přes OpenAI-style API, vlastní Python skript místo Claude Code 30
C CERIT přes Claude Code + LiteLLM proxy (překladač protokolů uprostřed) 10
D CERIT přímo přes Claude Code — stejné nástroje, jiný model na pozadí 20–100

Tracky A a D jsou nejsrovnatelnější — v obou případech běží Claude Code s celou sadou nástrojů (čtení souborů, spouštění příkazů, psaní kódu). Rozdíl je jen v tom, jaký model odpovídá. Track B testuje modely samotné bez Claude Code. Track C jsem zkoušel, ale nedoporučuji — viz níže.

Úloh bylo deset: psaní agentů pro Telegram a systemd, datové pipeline (RSS, TF-IDF), infra skripty, integrace LLM do kódu, REST klienti, zpracování PDF, vector databáze s RAG, orientace v cizím repozitáři, konfigurace prostředí a automatizace sociálních sítí. Každá úloha dostala čisté pracovní prostředí a sadu automatických testů.

Výsledek každého běhu se skládá ze čtyř složek, které se váženě průměrují do jednoho skóre 0–1: 

výsledné skóre = 0.40 × správnost + 0.25 × úplnost
               + 0.20 × kvalita kódu + 0.15 × hospodárnost tokenů

správnost = 0.20 × syntax + 0.40 × automatické testy + 0.40 × hodnocení LLM
hospodárnost = min(1.0, 15 000 / počet tokenů)
Jedno omezení dopředu: část hodnocení dělá sám Claude (jako „nezávislý hodnotič"). To je trochu konflikt zájmů — Claude hodnotí Claude. Tato složka tvoří přibližně 16 % celkového skóre. Složka automatických testů je na tom nezávislá a je proto spolehlivější.

Tady je zajímavost, která mě trochu překvapila: CERIT modely prošly více automatickými testy než Claude Sonnet. Gemma4 správnost 0.882 vs Sonnet 0.797. Tohle jsou pytestové testy bez závislosti na LLM hodnotiteli. CERIT modely jsou tedy v průměru funkčně správnější, jenže...

...Claude vede na čistotě kódu. Složka „kvalita kódu" (0.944 u Sonnetu vs 0.667 u CERIT modelů) hodnotí věci jako ošetření chyb, bezpečnostní vzory nebo čitelnost. Kód z nativního Claude je zkrátka uklizený. CERIT modely to dodělají, ale bez lesku.

Velký rozdíl je taky v hospodárnosti tokenů — Claude potřebuje na úspěšný běh okolo 4 000 tokenů, CERIT modely v Track D spíše 17 000–45 000. Jenže pro uživatele CERIT to nehraje roli, protože tokeny na CERIT nestojí nic.

Dotažení: číslo, které potřebuje vysvětlení

„Dotažení" znamená, že Claude Code vůbec úlohu dokončil — subprocess skončil bez fatální chyby a vytvořil soubory. Neznamená to, že výsledek byl dobrý. Ale bez dotažení nevznikne nic.

Proč Claude nedotahuje? Jde o strategii. Claude Sonnet typicky udělá 4–5 volání nástrojů (čtení, psaní, spuštění příkazu). Na jednoduché úlohy to stačí. Na zadání, kde se čekají čtyři výstupní soubory najednou (Dockerfile, docker-compose.yaml, .env a inicializační skript), to nestačí — Claude skončí dřív, než je vše hotové. CERIT modely dělají 10–14 kroků a úlohu spíše dodělají.

Prakticky vzato: pokud pouštíte agenta bez dozoru přes noc, záleží vám na tom, aby úlohy vůbec skončily. V tomto ohledu gemma4 a kimi-k2.6 na CERIT vedou.

CERIT modely to zvládají lépe díky tomu, že iterují přes více kroků a opravují se za chodu.

Kolik tokenů to celé stojí

Nativní Claude potřebuje na úspěšný běh asi 4 000 tokenů a 4–8 kroků nástrojů. CERIT modely v Track D potřebují 17 000–45 000 tokenů a 10–14 kroků. To je zhruba 4–10× více — ne stovkový řád, jak by se možná čekalo.

Větší čísla (až 540 000 tokenů na běh) se vyskytla v Track B s vlastním harnesem u některých modelů — to je ale důsledek toho, jak harness komunikuje, ne vlastnost modelu. Do Track D to nepřenášejte.

Pro uživatele CERIT je tahle úvaha stejně akademická — služba tokenový počet neúčtuje. Větší spotřeba tokenů proto nepředstavuje praktickou nevýhodu.

Bug, na který jsem narazil: thinker a deepseek-v3.2-thinking

Dva modely na CERIT — alias thinker a konkrétní model deepseek-v3.2-thinking — se přes Anthropic rozhraní chovaly divně. Claude Code s nimi v testech neudělal vůbec nic: žádné soubory, prázdný výsledek.

Po delším zkoumání jsem zjistil příčinu. DeepSeek modely mají vlastní formát pro volání nástrojů — používají speciální tokeny jako <｜DSML｜function_calls>. Jenže CERIT adaptér tyto tokeny nezpracovával správně a posílal je Claude Code jako „myšlenkový blok" místo jako skutečné volání nástroje. Claude Code pak čekal, ale příkaz ke spuštění nástroje nikdy nepřišel — a session skončila prázdná.

Přes OpenAI rozhraní (Track B) stejné modely fungují bez problémů a dosáhly skóre 0.728. Chyba je tedy specifická pro Anthropic-shape adaptér na CERIT. Nahlásil jsem to Lukáši Hejtmánkovi z CERIT v dubnu 2026.

Prakticky: Modely thinker a deepseek-v3.2-thinking přes Claude Code zatím nepoužívejte. Pokud chcete DeepSeek, funguje přes vlastní Python skript a OpenAI rozhraní.

Infrastruktura se vyvíjí

Testy probíhaly v dubnu 2026 — a to je důležité zmínit. CERIT LLM gateway se aktivně rozvíjí. Od té doby se mohly přidat nové modely, změnit verze stávajících, nebo se třeba opravila výše zmíněná chyba s DeepSeek tokeny. Výsledky tedy popisují konkrétní stav v konkrétním čase, ne trvalou pravdu.

Nabídka modelů na CERIT roste. V době testování bylo dostupných zhruba deset funkčních modelů přes Anthropic rozhraní, plus další přes OpenAI rozhraní. Jsou to modely z rodin Qwen, GLM, Kimi a Gemma — tedy open-source modely vyvíjené převážně v Číně, které ale na mezinárodních benchmarcích standardně obstojí a jsou pro akademické prostředí velmi zajímavou možností.

Track C (LiteLLM proxy): nedoporučuji

LiteLLM je nástroj, který překládá mezi různými API formáty. Myšlenka Track C byla: přesměrovat Claude Code na lokální proxy, která převede volání na CERIT formát. V praxi to ale většinou časovalo po 300 sekundách — modely prostě nestíhaly odpovědět v limitu, který Claude Code očekává. Na 10 trialech jsem nenasbíral data na srovnání. Pro běžné použití nedoporučuji.

Jak to nastavit v praxi

Nejjednodušší způsob je shell funkce v ~/.bashrc. Přepíše proměnné prostředí a spustí standardní claude: 

claude-cerit() {
  ANTHROPIC_BASE_URL="https://llm.ai.e-infra.cz/"  \
  ANTHROPIC_MODEL="gemma4"                          \
  ANTHROPIC_DEFAULT_OPUS_MODEL="gemma4"             \
  ANTHROPIC_DEFAULT_SONNET_MODEL="gemma4"           \
  ANTHROPIC_DEFAULT_HAIKU_MODEL="glm-5"             \
  MAX_THINKING_TOKENS=0                             \
  DISABLE_TELEMETRY=1                               \
  command claude "$@"
}

Pak stačí napsat claude-cerit místo claude a jedete na CERIT infrastruktuře.

U aliasů platí obecné upozornění: CERIT může alias přemapovat na jiný model bez upozornění — případ thinker to ukázal plasticky. Pro důležité nebo produkční úlohy používejte konkrétní názvy modelů, ne aliasy.

Co si z toho odnést

Gemma4 na CERIT dosáhla 96 % výkonu Claude Sonnet na škále 0–1 — a to zdarma, bez omezení tokenů, na české akademické infrastruktuře. Rozdíl je v detailech: Claude píše čistější, bezpečnější kód; CERIT modely projdou více automatických testů a spolehlivěji dotáhnou složitější úlohy do konce.

Pro rutinní programátorskou práci — psaní agentů, pipeline, skripty, orientace v repozitáři — je CERIT reálná alternativa. Na architekturální diskuse nebo delší složité session bych zatím sáhl po nativním Claude.

A ještě jednou: testy probíhaly v dubnu 2026. CERIT infrastruktura se aktivně vyvíjí a nabídka modelů roste. Stojí za to benchmark čas od času zopakovat — čísla se budou měnit, pravděpodobně k lepšímu.

Poděkování: Dík CERIT-SC za provoz LLM gateway a za to, že je přístupná pro celou českou akademii. Konkrétní dík Lukáši Hejtmánkovi za rychlou reakci na hlášení bugu s DeepSeek modely a zájem o spolupráci na reprodukci.
